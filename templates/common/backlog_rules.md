# Backlog Rules

## User Story Format
- As a [persona], I want [capability], so that [benefit].

## Acceptance Criteria
- [Given/When/Then]
- [Success and error scenarios]

## Definition of Ready
- [Sufficient context]
- [Dependencies mapped]
- [Acceptance criteria defined]

## Definition of Done
- [Code + tests]
- [Documentation updated]
- [Metrics / observability when applicable]
